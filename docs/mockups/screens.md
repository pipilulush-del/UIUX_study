# Mockup Screens — ContentsCollector

## Overview

각 화면의 목적, 레이아웃, 상태, 리뷰 질문을 상세 기술한다.
ASCII 도식은 정보 구조를 빠르게 파악하기 위한 것이며, 정확한 픽셀 배치가 아니다.
ID는 plan.md와 동일한 S001–S005를 사용한다.

## Global Mockup Assumptions

- **레이아웃:** 최대 콘텐츠 폭 ~1200px, 중앙 정렬, 좌우 패딩 48px (데스크톱 기준)
- **배경:** 전체 페이지 `#1f2228` (다크 캔버스), 단색 유지
- **헤더:** 고정(sticky), 로고(좌) + "내 저장 목록" 링크(우), 배경 동일
- **더미 데이터:** 오늘 날짜 2026.06.12, 콘텐츠 12개 기준, YouTube 4건·뉴스 5건·커뮤니티 3건 혼합
- **반응형 기준:** 데스크톱 우선(1024px+). 모바일(640px 미만) 참고 사항은 각 화면 Responsive Notes에 명시.
- **아이콘:** Lucide React — Bookmark(저장), BookmarkCheck(저장됨), Youtube, Newspaper, MessageSquare 등 사용

## Screen S001 - 홈 — 기본 랭킹 피드

**Purpose:** 오늘의 AI 핫 콘텐츠를 랭킹 순으로 한 화면에서 보여주는 메인 화면.

**Entry Points:**
- 브라우저에서 URL 직접 접속
- S004/S005에서 헤더 로고 클릭
- S003에서 필터 전체 해제

**Exit Points:**
- 카드 제목 클릭 → 외부 원문 (새 탭)
- 🔖 탭 → S002 overlay(토스트) 후 계속 S001
- 토픽·소스 탭 선택 → S002
- "내 저장 목록" 링크 클릭 → S004 또는 S005

**Primary User Action:** 랭킹 카드 훑기 → AI 요약 읽기 → 🔖 탭 또는 제목 클릭

**Secondary Actions:**
- 토픽 필터 탭 클릭 → S002로 전환
- 소스 타입 필터 탭 클릭 → S002로 전환
- 헤더 "내 저장 목록" 링크 클릭 → S004/S005

**Key UI Regions:**
- **Header:** 로고(ContentsCollector, Pretendard) + "내 저장 목록" 텍스트 링크(우측)
- **Intro Bar:** 서비스 한 줄 설명(text-secondary), 오늘 날짜 + 섹션 타이틀
- **Filter Tabs:** 토픽 탭 행 + 소스 타입 탭 행 — "전체"가 기본 활성
- **Ranking Feed:** 카드 목록, 상하 구분선(border-default), 무한 스크롤 또는 한 페이지
- **Card:** 랭킹 번호(GeistMono badge), 썸네일(YouTube만), 제목(링크), 소스 아이콘·타입, 토픽 태그, 조회수/좋아요, AI 요약(1–2줄), 🔖 버튼

**Content / Copy Notes:**
- 섹션 타이틀: "오늘의 AI 랭킹" (날짜와 같은 줄 또는 바로 아래)
- 랭킹 번호: GeistMono, badge-mono 스타일(border 0.2, 0px radius)
- 조회수 표기: "2.1M", "850K" — GeistMono 12px badge
- AI 요약: "~해요" 체, 1–2줄 이내, text-secondary
- 🔖 버튼 레이블: 저장 전 Bookmark 아이콘(윤곽선), 저장 후 BookmarkCheck 아이콘

**Data / Placeholder Needs:**
- 더미 제목 3–5개 (YouTube 2개 이상 포함), 실제 조회수처럼 보이는 숫자
- YouTube 썸네일 플레이스홀더 이미지 (회색 직사각형 또는 샘플 이미지)
- AI 요약 더미 문장 (20–50자 내외, 존댓말)

**States:**
- **Default:** 카드 12개 이하, 토픽·소스 "전체" 활성
- **Loading:** 헤더·필터 탭 렌더, 카드 영역에 스켈레톤(얇은 border-default 직사각형 5개)
- **Empty:** → S003 참조
- **Error:** → S003 참조
- **Saved:** 🔖 → 🔖✓(BookmarkCheck)로 즉시 전환, 토스트 오버레이(S002 상태로)

**Responsive Notes:**
- **Desktop (1024px+):** 카드 1컬럼, 최대 폭 1200px, 썸네일 우측 고정 또는 좌측 인라인
- **Mobile (<640px):** 패딩 24px, 필터 탭 가로 스크롤, 썸네일 카드 상단 배치(16:9)

## Screen S002 - 홈 — 필터 적용 피드

**Purpose:** 토픽 또는 소스 타입 필터가 활성화된 홈. 현재 조건(활성 필터 칩)을 명확히 보여주고, 필터된 랭킹 피드를 제공한다. 저장 토스트 오버레이 상태도 이 화면에서 확인한다.

**Entry Points:**
- S001에서 토픽 탭 클릭 (예: "LLM")
- S001에서 소스 타입 탭 클릭 (예: "YouTube")
- S002에서 다른 토픽·소스 탭 추가 선택

**Exit Points:**
- 활성 필터 칩 × 클릭 또는 "전체" 탭 클릭 → S001
- 카드 제목 클릭 → 외부 원문 (새 탭)
- 🔖 탭 → 저장 토스트 overlay (2초 후 자동 소멸)
- 토스트 "내 목록 보기 →" 클릭 → S004/S005
- 필터 결과 없음 → S003
- 헤더 "내 저장 목록" → S004/S005

**Primary User Action:** 필터 탭 선택 → 피드 인플레이스 갱신 → 카드 탐색

**Secondary Actions:**
- 🔖 탭 → 저장 토스트 표시
- 활성 필터 칩 × 클릭 → 필터 해제
- 추가 탭 선택 → 조합 필터 적용

**Layout ASCII:**

 │                                                                 │

 │                                                                 │
 │  활성 필터:  [ LLM  × ]  [ ▷ YouTube  × ]                      │  ← 활성 필터 칩
 │  ─────────────────────────────────────────────────────────      │

 │ │      │        │  ▷ YouTube  ·  #LLM  ·  조회수 2.1M       │   │
 │ │      │ 썸네일 │  GPT-5의 기능과 출시 일정이 내부 문서를    │   │

 │ │      └────────┘  ▷ YouTube  ·  #LLM  ·  조회수 740K       │   │

**Key UI Regions:**
- **Filter Tabs:** 활성 탭 강조(border-strong 또는 text-primary, inactive는 text-tertiary)
- **Active Filter Chips:** 활성 필터 조건 표시 — 토픽 칩, 소스 타입 칩 각각 × 해제 버튼 포함
- **Filtered Feed:** 조합 조건에 맞는 랭킹 카드 목록 (S001 카드와 동일 구조)
- **Toast Overlay:** 화면 하단 중앙 고정, "저장됐어요  내 목록 보기 →", 2초 후 자동 소멸, border-strong

**Content / Copy Notes:**
- 활성 필터 칩 텍스트: 토픽명 그대로("LLM", "로보틱스"), 소스명("YouTube", "뉴스", "커뮤니티")

- 활성 탭 시각 처리: 탭 텍스트 text-primary + border-bottom 또는 배경 surface-elevated

**Data / Placeholder Needs:**
- LLM × YouTube 조합 기준 더미 카드 2–3개
- 저장 토스트 오버레이 상태 표현

**States:**
- **Single Filter (토픽만):** 토픽 칩 1개, 소스 탭은 "전체 소스"
- **Combined Filter (토픽 × 소스):** 칩 2개, 필터된 카드 목록
- **Toast Overlay:** 🔖 탭 후 하단 토스트 표시
- **결과 없음:** → S003으로 전환

**Responsive Notes:**
- **Mobile:** 활성 필터 칩이 필터 탭 아래 줄에 가로 스크롤로 배치
- **Desktop:** 활성 필터 칩을 카드 목록 바로 위 별도 행에 표시

## Screen S003 - 홈 — 빈·오류 상태

**Entry Points:**
- S001에서 오늘 수집 없음 또는 수집·네트워크 오류 발생

**Exit Points:**

- 네트워크 오류: "다시 시도" → S001

**Primary User Action:** 메시지 읽기 → 필터 해제 또는 재시도

**Secondary Actions:**

- 헤더 "내 저장 목록" → S004/S005
