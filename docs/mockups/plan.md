# Mockup Plan — ContentsCollector

## Overview

ContentsCollector 프론트엔드 목업 스프린트의 화면 목록과 화면 간 관계를 정의한다.
이 서비스는 PC·아이패드 데스크톱 우선의 반응형 웹이며, 화면은 크게 두 영역(홈, 내 저장 목록)으로 나뉜다.
홈에는 기본 피드·필터 적용·빈 상태 세 가지 주요 상태가 있어 각각 별도 screen으로 다룬다.

## Source Context

- **PRD:** FR-001~FR-015 (P0·P1 전체 반영) — 홈 랭킹 피드, 토픽·소스 필터 조합, YouTube 썸네일, 북마크·저장 목록, 저장 토스트, 빈/오류 상태
- **User Flows:** 플로우 1(첫 방문·일별 탐색), 플로우 2(토픽·소스 필터 탐색), 플로우 3(내 저장 목록 재탐색) 전체 반영
- **Additional Context:** docs/DESIGN.md 반영 — 다크 캔버스(#1f2228), GeistMono 랭킹 배지, Pretendard UI 텍스트, 0px 라운드, 모노크롬 팔레트, Lucide React 아이콘

## Mockup Scope

| Screen ID | 화면 이름 | 상태 |
|-----------|-----------|------|
| S001 | 홈 — 기본 랭킹 피드 | 전체 토픽·전체 소스, 콘텐츠 있는 상태 |
| S002 | 홈 — 필터 적용 피드 | 토픽·소스 필터 조합 활성, 저장 토스트 오버레이 포함 |
| S003 | 홈 — 빈·오류 상태 | 필터 결과 없음 / 수집 오류 / 네트워크 오류 |
| S004 | 내 저장 목록 — 콘텐츠 있는 상태 | 저장 카드 목록, 저장 해제 가능 |
| S005 | 내 저장 목록 — 빈 상태 | 저장 항목 없음, 홈으로 이동 CTA |

저장 토스트는 독립 화면이 아니라 S002의 overlay 상태로 다룬다.

## Out of Scope

- 실제 API 연동, 백엔드 수집·랭킹·요약 로직
- 계정/로그인 화면 (v1 범위 외)
- 날짜별 아카이브 탐색
- 콘텐츠 검색 화면
- 알림 설정, 소스 추가 제안 화면
- 실제 로컬 스토리지 저장 로직 구현

## Screen Flow

```text
[브라우저 접속]
  → S001 홈 — 기본 랭킹 피드
      → S002 홈 — 필터 적용 피드  (토픽 탭 / 소스 타입 탭 선택)
          → S003 홈 — 빈·오류 상태  (필터 결과 없음 또는 수집 오류)
              → S001  (필터 해제)
      → S003 홈 — 빈·오류 상태  (오늘 수집 없음 / 네트워크 오류)
      → [외부 원문]  (카드 제목 클릭 → 새 탭)
      → S004 내 저장 목록 — 콘텐츠 있는 상태  (헤더 "내 저장 목록" 클릭 또는 토스트 "내 목록 보기")
          → S005 내 저장 목록 — 빈 상태  (모든 항목 해제 시)
              → S001  ("오늘 랭킹 보러 가기" CTA)
```

## [x] Screen S001 - 홈 — 기본 랭킹 피드

**Purpose:** 서비스의 메인 화면. 오늘의 AI 핫 콘텐츠를 랭킹 순으로 한 화면에서 파악할 수 있도록 제공한다.

**User Moment:** 플로우 1 ① — 아침에 접속해 "오늘 AI 뭐 핫해?"를 바로 확인하는 순간

**Primary Action:** 랭킹 카드 훑기 → AI 요약 읽기 → 🔖 탭으로 저장 또는 제목 클릭으로 원문 이동

**Leads To:** S002 (필터 탭 선택), S003 (오류/빈 상태), S004/S005 (헤더 링크), 외부 원문 (새 탭)

**States to Mock:** 기본(콘텐츠 있음), 로딩(스켈레톤 카드)

**Review Focus:**
- [ ] 헤더에서 서비스 정체성(로고 + 한 줄 설명)이 한눈에 들어오는가?
- [ ] 오늘 날짜와 "오늘의 AI 랭킹" 레이블이 콘텐츠 신선도를 충분히 전달하는가?
- [ ] 토픽 탭과 소스 타입 탭의 위계 관계가 시각적으로 명확한가?
- [ ] YouTube 카드(썸네일 있음)와 뉴스·커뮤니티 카드(텍스트 전용)의 혼재가 리듬을 해치지 않는가?
- [ ] 🔖 버튼이 작동 의미를 명확하게 전달하는가?

**Done When:**
- [ ] 헤더, 날짜/타이틀, 토픽 필터 탭, 소스 타입 필터 탭, 랭킹 카드 목록이 모두 포함된 화면을 리뷰할 수 있다.
- [ ] YouTube 카드(썸네일 포함)와 텍스트 전용 카드 두 타입이 동시에 표현된다.

## [x] Screen S002 - 홈 — 필터 적용 피드

**Purpose:** 토픽 또는 소스 타입 필터를 활성화한 상태의 홈. 활성 필터 칩이 피드 상단에 노출되어 현재 조건을 명확히 알린다. 저장 토스트 오버레이도 이 화면에서 확인한다.

**User Moment:** 플로우 2 — "LLM만 모아보고 싶다" — 토픽 탭 선택 직후 피드가 인플레이스로 갱신되는 순간

**Primary Action:** 토픽·소스 필터 탭 선택 → 필터된 피드 탐색 → 🔖 탭으로 저장(토스트 확인)

**Leads To:** S001 (필터 해제), S003 (결과 없음), S004/S005 (토스트 링크 또는 헤더)

**States to Mock:** 필터 1개 활성(토픽만), 필터 조합 활성(토픽 × 소스), 저장 토스트 오버레이

**Review Focus:**
- [ ] 활성 필터 칩이 현재 조건을 한눈에 파악하게 하는가?
- [ ] 필터 칩의 × 해제 버튼이 발견하기 쉬운가?
- [ ] 저장 토스트("저장됐어요 내 목록 보기 →")가 피드를 가리지 않고 적절한 위치에 나타나는가?
- [ ] 토픽 + 소스 타입 조합 필터가 YouTube를 차별 포인트로 명확하게 드러내는가?

**Done When:**
- [ ] 활성 필터 칩이 포함된 필터 조합 상태 화면을 리뷰할 수 있다.
- [ ] 저장 토스트 오버레이 상태가 표현된다.

## [x] Screen S003 - 홈 — 빈·오류 상태

**Purpose:** 필터 결과가 없거나, 오늘 콘텐츠 수집이 되지 않았거나, 네트워크/수집 오류가 발생했을 때의 홈 상태. 헤더·필터 탭은 그대로 유지되고 피드 영역에만 상태 메시지가 표시된다.

**User Moment:** 플로우 2 — 필터 조합에 해당하는 콘텐츠가 없을 때 / 플로우 1 — 아침에 수집 오류로 피드가 비어 있을 때

**Primary Action:** 상태 메시지 확인 → 필터 해제(× 또는 "전체" 탭) 또는 재시도

**Leads To:** S001 (필터 해제 또는 재시도)

**States to Mock:** 필터 빈 상태, 오늘 수집 없음(초기 운영), 수집 오류, 네트워크 오류

**Review Focus:**
- [ ] 빈 상태 메시지가 원인(필터 결과 없음 vs 수집 오류)을 구분해 전달하는가?
- [ ] 다음 행동(필터 해제, 재시도)을 자연스럽게 유도하는가?
- [ ] 메시지 톤이 DESIGN.md "calm, solution-first" 원칙에 맞는가?

**Done When:**
- [ ] 필터 빈 상태, 수집 오류 두 가지 이상의 상태 메시지 표현이 리뷰 가능하다.

## [ ] Screen S004 - 내 저장 목록 — 콘텐츠 있는 상태

**Purpose:** 북마크한 콘텐츠를 저장 날짜 최신순으로 모아보고, 원문으로 이동하거나 저장을 해제하는 화면.

**User Moment:** 플로우 3 ① — "저장해둔 거 다시 볼까" — 헤더 "내 저장 목록" 클릭 직후

**Primary Action:** 저장 카드 목록 탐색 → 제목 클릭으로 원문 이동 또는 🔖✓ 탭으로 저장 해제

**Leads To:** 외부 원문 (새 탭), S005 (마지막 항목 해제), S001 (헤더 로고 클릭)

**States to Mock:** 기본(카드 2–5개), 저장 해제 직후 항목 사라짐 애니메이션(선택)

**Review Focus:**
- [ ] 저장 수 표시("N개")가 목록 규모를 빠르게 파악하게 하는가?
- [ ] 저장 날짜 표시가 콘텐츠의 시간적 맥락을 충분히 전달하는가?
- [ ] 🔖✓ 버튼이 "이미 저장됨, 탭하면 해제" 상태를 명확하게 전달하는가?
- [ ] 홈(랭킹 피드)으로 돌아가는 경로가 명확한가?

**Done When:**
- [ ] 저장 카드 목록(썸네일, 제목, 소스, 토픽, 저장일, AI 요약, 🔖✓)이 표현된 화면을 리뷰할 수 있다.

- [ ] 페이지 전체가 너무 허전하거나 무겁지 않은가?

**Done When:**

## Open Questions

- 토픽 필터 탭과 소스 타입 필터 탭의 시각적 위계: 탭 두 줄을 어떻게 구분할지 (구분선? 크기 차이? 들여쓰기?)
- 랭킹 피드에서 YouTube 카드(썸네일 있음)와 텍스트 카드가 섞일 때 레이아웃 리듬 — 썸네일 크기를 고정 비율로 통일할지, 텍스트 카드와 높이를 맞출지
- 로딩 스켈레톤 표현 방식 — 카드 수를 고정(예: 5개)으로 할지, 마지막 피드 콘텐츠 수를 기억해 맞출지
- 저장 토스트 위치 — 화면 하단 중앙 vs 우하단 중 어느 쪽이 카드 조작을 방해하지 않는가?
- 데스크톱 최대 콘텐츠 폭(~1200px) 안에서 카드 레이아웃 — 1컬럼 전체 폭 vs 좌우 여백 비율
