## 1. 개요

- **한 줄 정의:** AI 씬을 깊이 팔로우하는 사람을 위해, YouTube·뉴스·커뮤니티에서 핫한 콘텐츠를 매일 자동 수집·요약·게시하는 AI 큐레이션 웹 서비스
- **해결하는 문제:** AI 분야 정보를 얻으려면 유튜브, aitimes 등 여러 소스를 직접 돌아다녀야 하고, 전문가들이 어디서 정보를 얻는지 몰라 좋은 소스 자체를 놓치는 경우가 많다.
- **타깃 사용자:** AI 분야를 꾸준히 팔로우하는 개인 — 특정 토픽(LLM, 로보틱스, AI 툴 등)을 깊게 파고들 때 최신 핫 콘텐츠를 효율적으로 수집하고 싶은 사람. PC·아이패드에서 여유 있게 탐색하는 스타일.
- **플랫폼:** 반응형 웹 (데스크톱 우선 — PC·아이패드)

---

## 2. 배경 & 근거

- **시장/경쟁 맥락:** AI 뉴스 큐레이션 시장은 이메일 뉴스레터(The Rundown AI, TLDR AI)가 장악하고 있으며, 모바일 앱(Particle)도 존재한다. 한국어 서비스(AI Trends KR)는 웹이지만 YouTube 소스와 인기도 기반 자동 랭킹이 없다. 모든 직접 경쟁사가 에디터 큐레이션 방식을 취한다.
- **우리의 차별 프레이밍:** "AI를 깊게 파는 사람이 매일 아침 열어보는 웹 보드 — YouTube·뉴스·커뮤니티를 한 화면에서 인기도 순으로." 두 가지 차별 포인트가 공백을 만든다: (1) YouTube 포함 멀티소스 — 경쟁사 중 YouTube를 수집 소스로 넣는 곳이 없다. (2) 인기도 기반 자동 랭킹 — 에디터 주관 없이 조회수·좋아요로 투명하게 선별한다.

---

## 3. 목표 (Goals)

1. 매일 아침 AI 씬의 핫 콘텐츠를 한 화면에서 파악할 수 있는 신뢰할 수 있는 일별 큐레이션 웹 보드를 제공한다.
2. YouTube를 포함한 멀티소스 자동 수집과 인기도 기반 랭킹으로 에디터 편향 없는 투명한 선별을 실현한다.
3. 토픽·소스 필터를 통해 특정 분야를 깊게 파고드는 사용자의 집중 탐색을 지원한다.

## 4. User Scenarios   ← userflow

### Scenario 1 — 첫 방문·일별 탐색 ("오늘 AI 뭐 핫해?")

- **사용자:** AI 씬을 매일 팔로우하는 개인
- **상황:** 아침에 접속해 오늘의 AI 핫 콘텐츠를 빠르게 훑고 싶음
- **목표:** 오늘 AI에서 핫한 콘텐츠를 파악하고, 나중에 볼 것을 저장하거나 바로 원문으로 이동
- **흐름:** 홈 진입 → 오늘 날짜 랭킹 피드 확인 → 카드 훑기 → AI 요약 읽기 → 🔖 탭으로 저장 또는 제목 클릭으로 원문 이동(새 탭)
- **완료 조건:** 오늘의 AI 핫 콘텐츠 목록이 랭킹 순으로 표시되고, 관심 콘텐츠에 대해 저장 또는 원문 이동 중 하나를 완료함

### Scenario 2 — 주제·소스 필터 탐색 ("LLM만 모아보고 싶다")

- **사용자:** AI 분야 중 특정 토픽(예: LLM)을 집중적으로 팔로우하는 사용자
- **상황:** 전체 피드에서 관심 토픽 콘텐츠만 빠르게 보고 싶음
- **목표:** 토픽 탭 하나로 관심 분야 콘텐츠만 집중 탐색
- **흐름:** 홈 진입 → 토픽 탭 선택(예: LLM) → 인플레이스 피드 갱신 → 필터된 랭킹 카드 탐색 → 필요 시 소스 타입 필터 추가 조합
- **완료 조건:** 선택한 토픽(또는 토픽×소스 타입 조합)의 랭킹 피드가 즉시 표시되고, 관심 콘텐츠에 대한 액션이 완료됨

### Scenario 3 — 내 저장 목록 재탐색 ("저장해둔 거 다시 볼까")

- **사용자:** 이전에 콘텐츠를 저장해둔 사용자
- **상황:** 아껴둔 콘텐츠를 나중에 다시 꺼내 보고 싶음
- **목표:** 저장한 콘텐츠를 한 곳에서 확인하고 원문으로 이동
- **흐름:** 헤더 '내 저장 목록' 클릭 → 저장된 카드 목록 확인 → 제목 클릭으로 원문 이동 또는 🔖✓ 탭으로 저장 해제
- **완료 조건:** 저장한 콘텐츠 목록이 저장 날짜 최신순으로 표시되고, 원문 이동 또는 저장 해제 액션이 완료됨

### Scenario 4 — 과거 날짜 탐색 ("어제(또는 그제) 뭐 핫했지?")

- **사용자:** 하루이틀 접속하지 못했거나, 특정 날짜의 흐름을 되짚어 보고 싶은 사용자
- **상황:** 오늘 피드를 보다가 놓친 지난 날짜의 랭킹을 확인하고 싶음
- **목표:** 홈을 떠나지 않고 하루 단위로 과거 랭킹을 오가며 확인
- **흐름:** 홈 진입(기본=오늘) → 날짜 네비게이터의 ◀(이전 날) 클릭 → 해당 날짜의 랭킹 피드로 인플레이스 갱신 → 필요 시 더 이전 날로 이동하거나 '오늘'로 복귀
- **완료 조건:** 선택한 과거 날짜의 랭킹 피드가 표시되고, 오늘로 다시 돌아올 수 있다. 미래 날짜로는 이동할 수 없다(오늘이 최신).

## 6. Success Criteria

- SC-001: 신규 사용자가 접속 후 30초 이내에 오늘의 AI 핫 콘텐츠 랭킹 목록을 확인할 수 있다.
- SC-002: 사용자가 토픽 탭을 탭한 후 2초 이내에 필터된 피드가 화면에 반영된다.
- SC-003: 각 콘텐츠의 AI 요약이 1~2줄 이내로 표시되어 추가 클릭 없이 내용을 파악할 수 있다.
- SC-004: 저장 액션 후 토스트 알림이 2초 이내 표시되고, 내 저장 목록에 해당 항목이 반영된다.
- SC-005: 매일 오전 콘텐츠 게시가 자동 완료되어 사용자가 당일 오전 중 새 랭킹을 확인할 수 있다. *(가설 — 게시 시간 기준 미확정, Assumptions 참조)*

## 7. Edge Cases

- **오늘 수집 콘텐츠 없음(수집 오류·초기 운영):** 피드 빈 상태 — "곧 오늘의 콘텐츠가 채워질 거예요." 표시
- **소스 수집 API 오류:** 에러 메시지 — "수집 중 문제가 생겼어요. 잠시 후 다시 시도해주세요." 표시
- **네트워크 연결 없이 접속:** 네트워크 에러 안내 — "네트워크 연결을 확인해주세요." 표시
- **필터 조합에 해당하는 콘텐츠 없음:** 빈 상태 — "이 분야에서 오늘 올라온 콘텐츠가 없어요." 표시
- **선택한 과거 날짜에 보존된 랭킹이 없음(서비스 시작 이전 등):** 빈 상태 — "이 날짜에는 수집된 콘텐츠가 없어요." 표시
- **오늘 날짜에서 다음 날(미래)로 이동 시도:** ▶ 버튼 비활성(disabled) 처리 — 오늘이 가장 최신임을 시각적으로 전달
- **내 저장 목록 빈 상태(저장 항목 없음):** "아직 저장된 콘텐츠가 없어요. 관심 있는 콘텐츠의 🔖를 탭하면 여기 모여요." + [오늘 랭킹 보러 가기] CTA
- **이미 저장된 콘텐츠를 다시 저장하려는 경우(중복 저장):** 이미 저장된 상태(🔖✓)로 표시 — 탭 시 저장 해제(토글)
- **YouTube 썸네일 로드 실패:** 썸네일 없이 텍스트 카드로 graceful fallback
- **AI 요약 생성 실패:** 요약 없이 제목·소스·조회수만 표시

## 9. 브랜드 & 디자인   ← brandvoice + DESIGN

- **서비스 이름:** ContentsCollector (콘텐츠콜렉터)
- **한 줄 무드:** 빠르게 훑되, 제대로 골라서, 믿고 볼 수 있게.
- **보이스 요약:** Sounds like a sharp researcher friend who keeps an impeccably organized shelf — speaks with data, not drama.
- **UI 카피 톤 가이드:**
  - **Do:** 수치·출처를 드러내 신뢰를 만든다 ("26개 소스에서 추린 상위 12개"). 선별 기준을 자연스럽게 녹인다. 짧은 문장으로 일한 티를 낸다. 존댓말(~해요체) 기본.
  - **Don't:** FOMO·불안 자극 금지 ("놓치면 후회해요", "지금 당장"). 과장 이모지 금지(🔥💥🤯). 막연한 미래 예언형 금지 ("AI가 세상을 바꿀 거예요"). 느낌표는 한 화면에 최대 1개.
- **시각 디자인 시스템:** → docs/DESIGN.md 참조 (다크 캔버스 `#1f2228`, GeistMono 디스플레이, Pretendard UI 텍스트, 0px 라운드, 모노크롬 팔레트, Lucide React 아이콘)

## 10. 범위 / 비범위 & 우선순위

- **v1 범위:**
  - P0 전체: 홈 랭킹 피드, 멀티소스 자동 수집(YouTube·뉴스·커뮤니티), 인기도 기반 자동 랭킹, AI 자동 요약, 원문 링크, 매일 자동 게시, 토픽별 자동 분류
  - P1 전체: 토픽 필터 탭, 소스 타입 필터 탭, 필터 조합, YouTube 썸네일, 북마크(저장), 내 저장 목록(조회·해제), 저장 토스트 알림, 일별 아카이브(홈 날짜 네비게이터로 과거 날짜 랭킹 조회)

- **Out of scope (지금 안 함):**
  - 계정/로그인, 서버 측 저장 동기화
  - 공개 플랫폼 전환, 구독·팔로우 기능
  - 특정 주제 알림(푸시·이메일) 설정
  - 독자 소스 추가 제안
  - 콘텐츠 검색 기능
  - 날짜 점프용 캘린더/아카이브 전용 화면 (v1은 홈 날짜 네비게이터의 하루 단위 이동만 — 먼 과거로 한 번에 점프하는 UI는 이후)

- **우선순위 묶음(제품 관점):**
  - **MVP (P0):** 홈 랭킹 피드 + 멀티소스 수집 + 인기도 랭킹 + AI 요약 + 매일 자동 게시 + 토픽 자동 분류
  - **v1 (P0 + 핵심 P1):** MVP + 토픽·소스 필터 + YouTube 썸네일 + 북마크(저장·내 저장 목록) + 일별 아카이브(홈 날짜 네비게이터)
  - **이후 (P2·확장):** 구독·팔로우, 알림 설정, 소스 추가 제안, 멀티 기기 동기화

*구현 마일스톤 로드맵은 vibe-build의 roadmap 스킬이 docs/ROADMAP.md에 작성한다.*

## 11. Assumptions

1. 수집 대상 소스는 사전 정의된 목록(YouTube AI 채널, aitimes 등 뉴스 사이트, Reddit r/artificial·r/MachineLearning, Hacker News)으로 한정하며, v1에서는 운영자가 관리한다.
2. 콘텐츠 수집 및 랭킹은 하루 1회 자동 실행(매일 오전 특정 시간)을 기본으로 한다.
3. 초기 버전은 로컬 스토리지 기반 북마크로, 서버 측 계정 시스템은 포함하지 않는다.
4. 서비스 UI 언어는 한국어이며, AI 요약은 원문 언어와 무관하게 항상 한국어로 제공한다. 콘텐츠 제목은 원문 언어를 유지한다.
5. 토픽 분류 카테고리(LLM, 로보틱스, AI 툴, 연구·논문)는 v1 런칭 시 고정 목록으로 시작한다.
6. 인증/로그인 기능은 v1 범위에 포함하지 않는다.
7. 피드에 노출하는 콘텐츠 수는 소스별 최대 5~10개, 전체 최대 약 12~20개로 한다.
8. 일별 아카이브는 게시된 날짜의 랭킹을 보존해 조회하는 것으로, 과거 데이터의 재랭킹·재요약은 하지 않는다(게시 시점 스냅샷). 보존 기간 상한은 v1에서 미정(오픈 이슈).

## 12. 오픈 이슈 / 리스크

- **YouTube Data API 쿼터·비용:** 하루 1회 수집으로도 API 쿼터를 초과할 경우 대비 전략 미정
- **뉴스 사이트 크롤링 허용 여부:** 대상 사이트별 robots.txt 및 이용약관 확인 필요
- ~~**AI 요약 언어:**~~ ✅ 결정됨 — 원문 언어와 무관하게 AI 요약은 항상 한국어로 제공한다.
- **랭킹 기준 세부 사항:** 조회수·좋아요 가중치, 게시 시간 최신성(최근 N일 내) 반영 여부 미정
- **소스 목록 관리 방식:** YouTube 채널 목록, 뉴스 사이트 목록을 어떻게 초기 구성하고 확장할지 미정
- **게시 시간 기준:** 매일 자동 게시 시각(KST 기준 오전 몇 시)이 미확정 (SC-005 가설과 연결)
- **모바일 UX 상세:** 데스크톱 우선 설계지만, 모바일 반응형 UX의 카드 레이아웃·필터 탭 접힘 방식은 아직 상세 정의되지 않음
